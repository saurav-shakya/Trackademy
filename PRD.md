# Product Requirements Document (PRD)

## 1. Product Name
Trackademy

---

## 2. Overview
A centralized platform to manage student activity approvals, track participation, handle reimbursements, and generate verified certificates while building a long-term student portfolio.

---

## 3. Problem Statement
Current process is:
- Manual (paper/email-based)
- Slow approvals
- No tracking visibility
- No centralized achievement record
- Risk of fake certificates

---

## 4. Solution
A digital workflow system that:
- Streamlines approvals
- Tracks status in real time
- Stores proof & achievements
- Generates verified certificates
- Builds a student portfolio

---

## 5. Goals
- Reduce approval time
- Improve transparency
- Digitize records
- Enable verifiable achievements
- Provide career-ready portfolios

---

## 6. User Roles

### Student
- Submit activity requests
- Track approval status
- Upload results
- Download certificates
- Manage profile

### Faculty
- Review requests
- Approve / Reject / Suggest changes

### HOD
- Department-level approval

### Admin
- Final approval
- Handle reimbursements
- Generate certificates

---

## 7. Core Features

### 7.1 Activity Submission
- Event name
- Organizer
- Date & location
- Participation type
- Budget details
- File uploads

---

### 7.2 Approval Workflow
Flow:
Student → Faculty → HOD → Admin

Statuses:
- Pending
- Approved
- Rejected
- Needs Revision

---

### 7.3 Real-Time Tracking
- Status timeline
- Notifications

---

### 7.4 Reimbursement System
- Expense submission
- Invoice upload
- Approval tracking

---

### 7.5 Achievement Upload
- Result type (winner/participant)
- Proof upload

---

### 7.6 Certificate Generation
- Auto-generated PDFs
- Unique ID
- QR verification

---

### 7.7 Student Portfolio
- Achievement timeline
- Shareable profile
- Exportable data

---

## 8. MVP Scope

### Included
- Submission system
- Approval workflow
- Status tracking
- File uploads
- Basic certificate generation

### Excluded (Future)

- Advanced analytics
